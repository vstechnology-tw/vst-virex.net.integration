Simulator 操作手冊 Simulator User Manual

使用 Virex.NET Integration Simulator 驗證 SDK 與通訊流程 Validate SDK and Protocol Behavior with the Virex.NET Integration Simulator

Simulator 是客戶端整合開發時的本機測試工具。它提供 REST、TCP 與 MQTT 端點， 並可模擬 WaferInfo、狀態變化、result summary 與 error event。 The simulator is the local test tool for customer-side integration development. It exposes REST, TCP, and MQTT endpoints and can simulate WaferInfo, state transitions, result summaries, and error events.

用途Purpose

驗證 SDKValidate the SDK

確認 VirexClient 可以讀取狀態、送出 WaferInfo、啟動流程並查詢 result summary。Confirm that VirexClient can read status, submit WaferInfo, start a cycle, and query result summaries.

驗證 raw protocolValidate Raw Protocols

讓非 C# 系統測試 REST payload、TCP/NDJSON frame 與 MQTT topic。Allow non-C# systems to test REST payloads, TCP/NDJSON frames, and MQTT topics.

模擬事件Simulate Events

在本機模擬 status、wafer-info、result、error，方便客戶端先完成處理邏輯。Simulate status, wafer-info, result, and error events locally so client handling logic can be completed early.

App UI 說明App UI

下圖是實際 Simulator App 介面。請依照標號了解各區用途與操作順序。 The image below is the actual Simulator App interface. Use the markers to understand each area and the operating order.

1 Connection Settings

設定 REST prefix、TCP port、MQTT host/port/topic，以及 result prefix。測試前先確認這區。Configure REST prefix, TCP port, MQTT host/port/topic, and result prefix. Confirm this area before testing.

2 WaferInfo

輸入 Lot ID、Wafer ID、Recipe ID、Slot、FOUP ID、Chamber ID。Enter Lot ID, Wafer ID, Recipe ID, Slot, FOUP ID, and Chamber ID.

3 State

顯示 initialized、processState 與 recipe，並提供主要操作按鈕。Shows initialized, processState, and recipe, and contains the main operation buttons.

4 Event Log

顯示 server 啟動、WaferInfo 更新、cycle、result、error 等執行紀錄。Shows server start, WaferInfo updates, cycle events, results, errors, and other activity.

5 Start Servers

第一個要按的按鈕。按下後 REST/TCP/MQTT 才會開始對外服務。The first button to press. REST/TCP/MQTT services are available only after this step.

6 Apply WaferInfo

修改 WaferInfo 後按下，讓 simulator 套用目前測試 wafer context。Press after editing WaferInfo so the simulator applies the current test wafer context.

7 Start Cycle

用來模擬完整 cycle，狀態會依序變化並產生 result summary。Simulates a full cycle, state transitions, and a result summary.

8 Emit Fake Result / Error

手動產生 result 或 error，適合測試客戶端事件處理與 UI 顯示。Manually emits a result or error to test client event handling and UI display.

標準操作 SOPStandard Operating Procedure

  1. 啟動 Simulator AppStart the Simulator App

    dotnet run --project src\Virex.NET.Simulator.WPF\Virex.NET.Simulator.WPF.csproj

  2. 確認連線設定Confirm Connection Settings

    第一次使用建議保留預設值：REST http://127.0.0.1:5088/、TCP 5089、MQTT 127.0.0.1:1883、topic Virex.NET。For first use, keep the defaults: REST http://127.0.0.1:5088/, TCP 5089, MQTT 127.0.0.1:1883, topic Virex.NET.

  3. 按下 Start ServersPress Start Servers

    成功後 Event Log 會顯示 REST listening、TCP listening 與 MQTT 啟動/連線紀錄，SDK 或 sample code 才能連線。When successful, Event Log shows REST listening, TCP listening, and MQTT started/connected records. SDK and sample clients can connect after this step.

  4. 先驗證 not_initializedVerify not_initialized First

    在尚未按 Initialize 前執行 SDK 或 REST sample。當 sample 呼叫 Start 時，預期回傳 HTTP 409 not_initialized。這表示 sample 正在正確反映 UI 狀態。Before pressing Initialize, run the SDK or REST sample. When the sample calls Start, the expected return is HTTP 409 not_initialized. This means the sample is correctly reflecting UI state.

  5. 輸入並套用 WaferInfoEnter and Apply WaferInfo

    填入 Lot ID、Wafer ID、Recipe ID、Slot、FOUP ID、Chamber ID，按 Apply WaferInfo，或讓 sample 送出 WaferInfo。Event Log 應同一行列出所有欄位。Enter Lot ID, Wafer ID, Recipe ID, Slot, FOUP ID, and Chamber ID, then press Apply WaferInfo, or let a sample send WaferInfo. Event Log should list all fields in one line.

    WaferInfo updated from REST: lotId=LOT-RAW-REST-001, waferId=W01, recipeId=RCP-A, slot=1, foupId=FOUP-A, chamberId=CH-1

  6. 執行 Initialize 與 Start CycleRun Initialize and Start Cycle

    按 Initialize 後狀態變成 initialized，再按 Start Cycle 模擬 capturing、inspecting、saving 等流程。Press Initialize to set initialized state, then press Start Cycle to simulate capturing, inspecting, and saving states.

  7. 產生測試事件Emit Test Events

    使用 Emit Fake Result 測試 result handling；使用 Emit Error 測試 error handling。MQTT sample 執行中時，這兩個按鈕應分別產生 Virex.NET/result 與 Virex.NET/error。Use Emit Fake Result to test result handling and Emit Error to test error handling. While an MQTT sample is running, these buttons should produce Virex.NET/result and Virex.NET/error.

  8. 結束測試End the Test

    測試完成後按 Stop Servers，或直接關閉視窗讓 simulator 停止服務。After testing, press Stop Servers or close the window to stop simulator services.

疑難排解Troubleshooting

  • Start Servers 失敗Start Servers fails
    檢查 REST prefix 是否被占用、TCP port 是否已被其他程式使用、MQTT broker 是否可連線。Check whether the REST prefix or TCP port is already in use and whether the MQTT broker is reachable.
  • SDK 連不上SDK cannot connect
    確認 Simulator 已按 Start Servers，且 SDK 的 RestBaseUrl、TCP host/port、MQTT host/port/topic 與 UI 一致。Confirm Start Servers was pressed and SDK RestBaseUrl, TCP host/port, and MQTT host/port/topic match the UI.
  • 看到 not_initializedSeeing not_initialized
    若尚未按 Initialize，這是預期結果。按 Initialize 並確認 Status 後再繼續 sample。If Initialize has not been pressed, this is expected. Press Initialize, confirm Status, then continue the sample.
  • 沒有收到 eventNo events received
    先看 Event Log 是否有產生事件，再確認 client 是否已訂閱 TCP 或 MQTT。First check Event Log for emitted events, then confirm the client is subscribed through TCP or MQTT.
  • 查不到 resultNo result returned
    先按 Start Cycle 或 Emit Fake Result，再確認查詢 filter 與目前 WaferInfo 一致。Run Start Cycle or Emit Fake Result first, then confirm query filters match the current WaferInfo.
