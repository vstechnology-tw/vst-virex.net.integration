REST API Reference

REST 適合 command/query：讀取狀態、送出 WaferInfo、控制流程與查詢 result summary。 REST is used for command/query flows: reading status, submitting WaferInfo, controlling the cycle, and querying result summaries.

Base URLBase URL

http://127.0.0.1:5088

常用流程Typical Flow

先 GET /api/status，再 POST /api/wafer-info，按需求 initialize/start，最後查詢 results。 Start with GET /api/status, submit POST /api/wafer-info, initialize/start as needed, then query results.

方法Method Route 用途Purpose
GET /api/status 讀取 initialized、processState 與 recipe。Read initialized, processState, and recipe.
GET /api/error 讀取目前 error 狀態。Read the current error state.
GET /api/wafer-info 讀取目前 WaferInfo。Read the current WaferInfo.
POST /api/wafer-info 送出 lot、wafer、recipe、slot、FOUP、chamber context。Submit lot, wafer, recipe, slot, FOUP, and chamber context.
POST /api/control/initialize, /terminate, /start, /stop 控制 simulator 狀態流程。Control the simulator state flow.
GET /api/results 查詢固定大小的 result summary。Query fixed-size result summaries.

Results

GET /api/results 支援 lotId、waferId、recipeId query string。 多個參數會以 AND 合併。Result response 只提供 summary，不包含 defect list、die list、crop list 或 image binary。 GET /api/results supports lotId, waferId, and recipeId query strings. Multiple parameters are combined with AND. Result responses are summary-only and do not include defect lists, die lists, crop lists, or image binaries.

TCP Socket Protocol

TCP 使用單一 socket 與 NDJSON framing。每一筆 frame 是一個 JSON object，必須以換行結尾。 TCP uses one socket with NDJSON framing. Each frame is one JSON object and must end with a newline.

FramingFraming

one JSON object per line
each frame ends with \n
UTF-8 encoding

Client 連到 Virex.NET-compatible service；同一條連線可送 inbound command，也可接收 outbound event。 The client connects to the Virex.NET-compatible service. The same connection can send inbound commands and receive outbound events.

MQTT Events

MQTT 只用於 outbound event，不用於 command/control。Simulator 按下 Start Servers 後會啟動 embedded broker。 MQTT is outbound-only and is not used for command/control. The simulator starts an embedded broker after Start Servers is pressed.

127.0.0.1:1883

MQTT payload 使用與 REST/TCP event 相同的 JSON shape；因為 child topic 已識別事件類型，payload 不需要 type 欄位。 MQTT payloads use the same JSON shapes as REST/TCP events. The payload does not require a type field because the child topic identifies the event.

Protocol Versioning

公開套件與 protocol change 使用 semantic versioning，並保持 simulator、SDK、docs 與 contract tests 同步。 Public packages and protocol changes use semantic versioning, keeping simulator, SDK, docs, and contract tests aligned.

版本Version 意義Meaning 範例Example
1.0.0 初始公開 contract。Initial public contract. Initial REST/MQTT/TCP integration contract.
1.1.0 向後相容的欄位或 endpoint 新增。Backward-compatible field or endpoint addition. Backward-compatible field or endpoint added.
2.0.0 Breaking change。Breaking change. Breaking payload, route, topic, or behavior change.
  • Minor versionMinor Version
    不可移除或改名公開 JSON 欄位；可以加入 additive fields。Do not remove or rename public JSON fields; additive fields are allowed.
  • Major versionMajor Version
    Payload、route、topic 或行為的 breaking change 需要 major version。Breaking payload, route, topic, or behavior changes require a major version.
  • 同步原則Alignment Rule
    Simulator、SDK、docs 與 contract tests 必須描述同一個 protocol version。Simulator, SDK, docs, and contract tests must describe the same protocol version.
