產品定位 Product Purpose

Virex.NET Integration Kit 是提供給客戶整合 Virex.NET 相容系統的公開 SDK 與通訊範例。 它不是內部系統原始碼，而是用來協助外部系統建立穩定通訊的產品套件。 Virex.NET Integration Kit is a public SDK and communication sample package for customer systems integrating with Virex.NET-compatible services. It is not private application source code; it is a product package for building stable external communication.

SDK

Virex.NET.Client

提供 C# client wrapper，可操作 REST command/query，並訂閱 TCP 與 MQTT 事件。 C# client wrappers for REST commands and queries, plus TCP and MQTT event subscriptions.

Samples

samples

可直接執行的 SDK、raw REST、raw TCP 範例，用來確認環境與整合流程。 Runnable SDK, raw REST, and raw TCP examples for verifying environment setup and integration flow.

Simulator

Virex.NET.Simulator.WPF

本機模擬服務，提供 REST、TCP、MQTT 端點讓客戶在接正式系統前先驗證程式。 A local simulator exposing REST, TCP, and MQTT endpoints for validation before connecting to a production-compatible service.

快速開始 Quick Start

建議第一次使用時先跑 simulator 與 C# SDK sample，確認開發環境、端點與基本流程都正常。 For first-time use, run the simulator and the C# SDK sample to confirm the development environment, endpoints, and basic flow.

  1. 建置並測試套件 Build and Test the Kit 

    dotnet test Virex.NET.Integration.slnx

  2. 啟動 simulator Start the Simulator 

    dotnet run --project src\Virex.NET.Simulator.WPF\Virex.NET.Simulator.WPF.csproj

    需要 App UI 與操作 SOP 時，請開啟 Simulator 操作手冊。 Open the simulator manual for the app UI guide and operating SOP.

  3. 執行 SDK 範例程式 Run the SDK Sample 

    dotnet run --project samples\csharp-sdk\CSharpSdkSample.csproj

  4. 確認預期結果 Confirm Expected Results

    範例程式會讀取狀態、送出 WaferInfo、啟動流程，並查詢 result summary。若這些步驟成功， 代表 SDK 與 simulator 的基本通訊已建立。 The sample reads status, sends WaferInfo, starts a cycle, and queries result summaries. If these steps succeed, basic SDK-to-simulator communication is working.

SDK 使用方法 SDK Usage

C# 應用程式建議使用 VirexClient。它集中管理 REST、TCP、MQTT 設定， 並提供常用操作方法。 C# applications should start with VirexClient. It centralizes REST, TCP, and MQTT settings and exposes the common operations used by integrations. 

using Virex.NET.Client;
using Virex.NET.Contracts;

using var client = new VirexClient(new VirexClientOptions
{
    RestBaseUrl = "http://127.0.0.1:5088",
    TcpHost = "127.0.0.1",
    TcpPort = 5089,
    MqttHost = "127.0.0.1",
    MqttPort = 1883,
    MqttTopic = "Virex.NET",
    TimeoutMs = 5000,
});

var status = await client.GetStatusAsync();

await client.SetWaferInfoAsync(new WaferInfo
{
    LotId = "LOT-001",
    WaferId = "W01",
    RecipeId = "RCP-A",
    Slot = "1",
    FoupId = "FOUP-A",
    ChamberId = "CH-1",
});

await client.InitializeAsync();
await client.StartAsync();

var results = await client.QueryResultsAsync(lotId: "LOT-001");

主要方法 Main Methods

GetStatusAsync, SetWaferInfoAsync, InitializeAsync, TerminateAsync, StartAsync, StopAsync, QueryResultsAsync

錯誤處理 Error Handling

REST 回應非成功狀態時會丟出 VirexClientException，包含 HTTP status code 與 response body。 Non-success REST responses throw VirexClientException with the HTTP status code and response body.

典型使用情境 Typical Use Cases

由 MES / Host 下 WaferInfo Send WaferInfo from MES or Host

在 inspection cycle 前送出 lot、wafer、recipe、slot、FOUP、chamber 等資訊。 Send lot, wafer, recipe, slot, FOUP, and chamber context before an inspection cycle.

控制流程 Control the Cycle

使用 REST 或 TCP command 執行 initialize、start、stop、terminate。 Use REST or TCP commands for initialize, start, stop, and terminate operations.

接收事件 Receive Events

透過 TCP 或 MQTT 監聽 status、wafer-info、result、error 事件。 Listen for status, wafer-info, result, and error events through TCP or MQTT.

查詢結果摘要 Query Result Summaries

使用 REST 依 lotId、waferId、recipeId 查詢固定大小的 result summary。 Use REST to query fixed-size result summaries by lotId, waferId, and recipeId.

介面選擇指南 Interface Selection Guide

若使用 C#，請優先使用 SDK。若不是 C#，再依系統需求選擇 REST、TCP 或 MQTT。 Use the SDK first for C# applications. For other platforms, choose REST, TCP, or MQTT based on system needs.

疑難排解 Troubleshooting

REST 連不上REST Request Fails

確認 simulator 已啟動、RestBaseUrl 正確、防火牆允許連線，並查看 VirexClientException 的 response body。Confirm the simulator is running, RestBaseUrl is correct, firewall access is allowed, and inspect the VirexClientException response body.

TCP 沒收到事件TCP Events Are Missing

確認 host/port、NDJSON 每筆 frame 以換行結尾，並保持 event loop 執行。Confirm host/port, ensure each NDJSON frame ends with a newline, and keep the event loop running.

MQTT 沒收到事件MQTT Events Are Missing

確認 broker、port 1883、base topic Virex.NET 與訂閱 topic tree。Confirm broker connectivity, port 1883, base topic Virex.NET, and the subscribed topic tree.

查不到 resultResult Query Is Empty

確認 simulator 已產生 result，且 lotId、waferId、recipeId filter 與送出的 WaferInfo 一致。Confirm the simulator has emitted a result and that lotId, waferId, and recipeId filters match the submitted WaferInfo.
